# Glyphs API

The **Planisfy Glyphs API** serves font glyphs as Protocol Buffer (PBF) files for rendering text labels on maps. Our fonts use **Signed Distance Field (SDF)** rendering, which enables crisp, resolution-independent text at any zoom level.

<Callout type="info" title="What Are Glyphs?">
  Glyphs are individual character shapes from fonts, stored as SDF data in PBF
  format. MapLibre/Mapbox use SDF rendering for sharp text that scales smoothly
  without pixelation.
</Callout>

---

## Endpoint Overview

```
https://api.planisfy.com/fonts/v1/{fontstack}/{range}.pbf
```

The glyphs URL in a style is a **URL template** that MapLibre/Mapbox automatically expands:

| Token         | Description                 | Example                              |
| ------------- | --------------------------- | ------------------------------------ |
| `{fontstack}` | Font family and weight      | `Open Sans Regular`, `Roboto Medium` |
| `{range}`     | Character range (256 chars) | `0-255`, `256-511`, `65536-65791`    |

For example, with `"glyphs": "https://api.planisfy.com/fonts/v1/{fontstack}/{range}.pbf"`:

- `https://api.planisfy.com/fonts/v1/Open Sans Regular/0-255.pbf` — ASCII characters
- `https://api.planisfy.com/fonts/v1/Open Sans Regular/256-511.pbf` — Latin-1 Supplement
- `https://api.planisfy.com/fonts/v1/Noto Sans Arabic/65536-65791.pbf` — Arabic characters

---

## Request Parameters

| Component   | Description                                                  |
| ----------- | ------------------------------------------------------------ |
| `fontstack` | URL-encoded font name (e.g., `Open%20Sans%20Regular`)        |
| `range`     | Character range in format `{start}-{end}` (multiples of 256) |

### Character Ranges

Glyphs are served in blocks of 256 characters for efficient loading:

| Range           | Characters Included                      |
| --------------- | ---------------------------------------- |
| `0-255`         | ASCII (basic Latin, numbers, symbols)    |
| `256-511`       | Latin-1 Supplement (accented characters) |
| `512-767`       | Latin Extended-A                         |
| `768-1023`      | Combining diacritics                     |
| `...`           | ...                                      |
| `65536-65791`   | Arabic                                   |
| `131072-131327` | Emoji                                    |
| `...`           | ...                                      |

MapLibre only requests ranges needed for visible text labels.

## Authentication

Published glyph assets are normally loaded as public map assets:

```bash
GET https://api.planisfy.com/fonts/v1/Open%20Sans%20Regular/0-255.pbf
```

If your deployment protects glyph reads, send the API key with `X-API-Key`:

```bash
GET https://api.planisfy.com/fonts/v1/Open%20Sans%20Regular/0-255.pbf
X-API-Key: pk_your_key_here
```

<Callout type="warn" title="Self-Hosted Mode">
  When self-hosting, fonts are typically served publicly without authentication.
  Configure your tile server to serve fonts from a public directory.
</Callout>

## Example Requests

### cURL

```bash
# Fetch ASCII glyphs for Open Sans Regular
curl "https://api.planisfy.com/fonts/v1/Open%20Sans%20Regular/0-255.pbf" \
  --output glyphs.pbf

# Fetch Arabic glyphs
curl "https://api.planisfy.com/fonts/v1/Noto%20Sans%20Arabic/65536-65791.pbf" \
  --output arabic.pbf
```

### JavaScript (MapLibre GL JS)

Glyphs are automatically loaded when you use a Planisfy style:

```javascript
const map = new maplibregl.Map({
  container: "map",
  style: "https://api.planisfy.com/styles/v1/planisfy/basic",
});

// Glyphs are automatically fetched from:
// https://api.planisfy.com/fonts/v1/{fontstack}/{range}.pbf
```

### JavaScript (Inspecting Glyph Requests)

```javascript
// Monitor glyph loading
map.on("data", (e) => {
  if (e.sourceDataType === "metadata") {
    console.log("Glyphs loaded:", map.style.glyphManager);
  }
});
```

### Python

```python
import requests

fontstack = "Open Sans Regular"
range_start = 0
range_end = 255

url = f"https://api.planisfy.com/fonts/v1/{fontstack}/{range_start}-{range_end}.pbf"
headers = {"X-API-Key": "pk_your_key_here"}

response = requests.get(url, headers=headers)

if response.status_code == 200:
    with open("glyphs.pbf", "wb") as f:
        f.write(response.content)
    print(f"Saved {len(response.content)} bytes of glyph data")
```

## Response Format

### PBF (Protocol Buffer) Format

The API returns glyphs in **Protocol Buffer format**:

```
Content-Type: application/x-protobuf
Content-Encoding: gzip
Cache-Control: public, max-age=31536000
Access-Control-Allow-Origin: *
```

### Glyph Structure

Each glyph in the PBF contains:

| Field     | Type      | Description            |
| --------- | --------- | ---------------------- |
| `id`      | `integer` | Unicode codepoint      |
| `bitmap`  | `bytes`   | SDF bitmap data        |
| `width`   | `integer` | Glyph width in pixels  |
| `height`  | `integer` | Glyph height in pixels |
| `left`    | `integer` | Left bearing           |
| `top`     | `integer` | Top bearing            |
| `advance` | `integer` | Advance width          |

### SDF (Signed Distance Field) Rendering

SDF glyphs store distance values rather than pixel colors:

```
Value  = 0     → On the glyph edge
Value  < 0     → Inside the glyph
Value  > 0     → Outside the glyph
```

This enables:

- **Crisp edges** at any zoom level
- **Smooth scaling** without pixelation
- **Outline effects** (halos, strokes)
- **Dynamic coloring** via shaders

<Callout type="info" title="SDF Resolution">
  Glyphs are typically rendered at 24×24 to 48×48 SDF resolution. The GPU
  samples this data and applies a threshold to produce the final text.
</Callout>

## Using Glyphs in Styles

### Setting the Glyphs Property

```json
{
  "version": 8,
  "name": "My Style",
  "glyphs": "https://api.planisfy.com/fonts/v1/{fontstack}/{range}.pbf",
  "sources": { ... },
  "layers": [
    {
      "id": "labels",
      "type": "symbol",
      "source": "places",
      "layout": {
        "text-field": ["get", "name"],
        "text-font": ["Open Sans Regular"],
        "text-size": 12
      }
    }
  ]
}
```

### Text Font Configuration

```javascript
// Set a specific font
map.setLayoutProperty("labels", "text-font", ["Open Sans Regular"]);

// Use multiple fonts (fallback chain)
map.setLayoutProperty("labels", "text-font", [
  "Noto Sans Arabic Regular",
  "Open Sans Regular",
]);

// Change font weight based on zoom
map.setLayoutProperty("labels", "text-font", [
  "step",
  ["zoom"],
  ["Open Sans Regular"],
  12,
  ["Open Sans Medium"],
  14,
  ["Open Sans Bold"],
]);
```

### Text Field with Glyphs

```javascript
// Basic text label
map.setLayoutProperty("labels", "text-field", ["get", "name"]);

// Multi-line text
map.setLayoutProperty("labels", "text-field", [
  "format",
  ["get", "name"],
  { "font-scale": 1.2 },
  "\n",
  ["get", "description"],
  { "font-scale": 0.8 },
]);

// Icon + text combination
map.setLayoutProperty("poi-labels", "text-field", [
  "format",
  ["image", ["get", "icon"]],
  "\n",
  ["get", "name"],
]);
```

## Caching

Glyphs are cached very aggressively since they never change:

| Cache Type         | Duration | Description               |
| ------------------ | -------- | ------------------------- |
| **Browser/Client** | 1 year   | Long-term local cache     |
| **CDN/Edge**       | 1 year   | Cloudflare edge locations |
| **Origin**         | 1 year   | Planisfy servers          |

<Callout type="tip" title="Cache Efficiency">
  Since fonts are static, the 1-year cache is safe. Use versioned URLs if you
  ever update fonts:
  <br />
  `"glyphs": ".../v1/{fontstack}/{range}.pbf"`
</Callout>

| Plan       | Requests per Minute | Requests per Day |
| ---------- | ------------------- | ---------------- |

## Error Codes

| HTTP Code | Error             | Description                           |
| --------- | ----------------- | ------------------------------------- |
| `400`     | Bad Request       | Invalid fontstack or range format     |
| `401`     | Unauthorized      | Missing or invalid API key            |
| `403`     | Forbidden         | API key suspended or over quota       |
| `404`     | Not Found         | Font or character range not available |
| `429`     | Too Many Requests | Rate limit exceeded                   |
| `500`     | Server Error      | Internal server error                 |

### Error Response Format

```json
{
  "message": "Font not found: Unknown Font",
  "code": "font_not_found",
  "request_id": "req_abc123xyz"
}
```

## Self-Hosting Glyphs

When self-hosting Planisfy, you can serve your own font glyphs:

### Generating PBF Glyphs

Use **node-fontnik** to generate SDF PBF glyphs from TTF/OTF fonts:

```bash
# Install fontnik
npm install -g fontnik

# Generate glyphs for a font
fontnik generate \
  --font /path/to/OpenSans-Regular.ttf \
  --output /path/to/glyphs/ \
  --start 0 \
  --end 65535
```

### Directory Structure

```
glyphs/
├── Open Sans Regular/
│   ├── 0-255.pbf
│   ├── 256-511.pbf
│   ├── 512-767.pbf
│   └── ...
├── Noto Sans Arabic Regular/
│   ├── 65536-65791.pbf
│   └── ...
└── Roboto Medium/
    ├── 0-255.pbf
    └── ...
```

### Martin Configuration

Configure Martin to serve glyph files:

```yaml
# martin-config.yaml
fonts:
  - id: "planisfy"
    path: "/path/to/glyphs"
    url_prefix: "https://your-domain.com/fonts"
```

Access your glyphs at:

```
https://your-domain.com/fonts/Open%20Sans%20Regular/0-255.pbf
```

### Updating Your Style

Reference your custom glyphs in the style JSON:

```json
{
  "version": 8,
  "glyphs": "https://your-domain.com/fonts/{fontstack}/{range}.pbf",
  "sources": { ... },
  "layers": [ ... ]
}
```

<Callout type="warn" title="File Size Considerations">
  Full Unicode coverage (all scripts) requires hundreds of MB. Consider serving
  only the scripts you need to reduce bandwidth and storage.
</Callout>

## Performance Optimization

### Minimize Font Families

Use fewer font stacks to reduce glyph requests:

```javascript
// ❌ Too many fonts
layout: {
  'text-font': ['Open Sans Regular', 'Roboto Regular', 'Noto Sans Regular']
}

// ✅ One preferred font
layout: {
  'text-font': ['Open Sans Regular']
}
```

### Restrict Character Ranges

If your data uses a limited character set, preload specific ranges:

```javascript
// Preload ASCII only
map.style.loadGlyphs("Open Sans Regular", "0-255");
```

### Use Font Subsetting

For self-hosted fonts, subset TTF files to only needed glyphs before generating PBF:

```bash
# Subset font to Latin only
pyftsubset OpenSans-Regular.ttf \
  --output-file=OpenSans-Latin.ttf \
  --unicodes=U+0020-007F
```

## Switching from Mapbox

Replace your Mapbox glyphs URL:

### Before (Mapbox)

```json
{
  "glyphs": "mapbox://fonts/mapbox/{fontstack}/{range}.pbf"
}
```

### After (Planisfy)

```json
{
  "glyphs": "https://api.planisfy.com/fonts/v1/{fontstack}/{range}.pbf"
}
```

Or in code:

```javascript
const map = new mapboxgl.Map({
  container: 'map',
  style: {
    version: 8,
    glyphs: 'https://api.planisfy.com/fonts/v1/{fontstack}/{range}.pbf',
    sources: { ... },
    layers: [ ... ]
  }
});
```

<Callout type="tip" title="Font Compatibility">
  Planisfy uses <code>Open Sans</code> and <code>Noto Sans</code> fonts, which
  are open-source alternatives to Mapbox's proprietary fonts. Map styles should
  render correctly with minimal adjustments.
</Callout>
