# Tilequery API

The **Planisfy Tilequery API** allows you to retrieve data about specific features from vector tilesets, based on a given latitude and longitude. Our API is drop-in compatible with Mapbox Tilequery v1, powered by open-source vector tile technology.

<Callout type="info" title="Data Source">
Tilequery queries are powered by <a href="https://maplibre.org/martin/" target="_blank">Martin</a>, serving vector tiles from PMTiles or PostgreSQL with PostGIS.
</Callout>

---

## Endpoint Overview

```
GET https://api.planisfy.com/v4/{tileset_id}/tilequery/{lon},{lat}.json
```

| Component | Description |
|-----------|-------------|
| `tileset_id` | The ID of the map/tileset being queried. Use a comma-separated list for up to 25 tilesets |
| `{lon},{lat}` | The longitude and latitude coordinates to query |

---

## Request Parameters

### Required Parameters

| Parameter | Type | Description |
|-----------|------|-------------|
| `tileset_id` | `string` | The ID of the tileset to query. To query multiple tilesets, use comma-separated values (max 25) |
| `{lon},{lat}` | `number` | The longitude and latitude coordinates to query |

### Optional Parameters

#### Vector Tileset Parameters

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `radius` | `number` | `0` | Distance in meters to search for features. `0` = point-in-polygon query. Required for point/line features |
| `limit` | `integer` | `5` | Number of features to return (1-50). Returns closest features first |
| `dedupe` | `boolean` | `true` | Deduplicate features with same layer, geometry type, ID, and properties |
| `geometry` | `string` | `all` | Filter by geometry type: `polygon`, `linestring`, or `point` |
| `layers` | `string` | `all` | Comma-separated list of layers to query (skips non-existent layers) |

<Callout type="tip" title="Radius Behavior">
When using a radius, queries search tiles at the tileset's maximum zoom level, including the intersecting tile plus eight surrounding tiles. Large radii with large point/line datasets may not return all possible features.
</Callout>

## Example Requests

### cURL

#### Point-in-Polygon Query

```bash
# Query which polygon contains a point
curl "https://api.planisfy.com/v4/mapbox.places/tilequery/-122.4194,37.7749.json?access_token=pk.your-key-here"
```

#### Radius Query

```bash
# Find features within 100 meters
curl "https://api.planisfy.com/v4/mapbox.places/tilequery/-122.4194,37.7749.json?radius=100&access_token=pk.your-key-here"
```

#### Multiple Tilesets

```bash
# Query multiple tilesets at once
curl "https://api.planisfy.com/v4/mapbox.places,mapbox.streets/tilequery/-122.4194,37.7749.json?access_token=pk.your-key-here"
```

#### Layer Filtering

```bash
# Query specific layers only
curl "https://api.planisfy.com/v4/mapbox.places/tilequery/-122.4194,37.7749.json?layers=poi,landuse&access_token=pk.your-key-here"
```

#### Geometry Filtering

```bash
# Return only polygons
curl "https://api.planisfy.com/v4/mapbox.buildings/tilequery/-122.4194,37.7749.json?geometry=polygon&access_token=pk.your-key-here"
```

### JavaScript (Fetch)

```javascript
// Point-in-polygon query
async function queryPointInPolygon(lng, lat) {
  const response = await fetch(
    `https://api.planisfy.com/v4/mapbox.places/tilequery/${lng},${lat}.json?access_token=YOUR_KEY`
  );
  const data = await response.json();
  return data;
}

// Radius query
async function queryNearbyFeatures(lng, lat, radiusMeters = 100) {
  const response = await fetch(
    `https://api.planisfy.com/v4/mapbox.places/tilequery/${lng},${lat}.json?radius=${radiusMeters}&limit=10&access_token=YOUR_KEY`
  );
  const data = await response.json();
  return data;
}

// Query specific layers
async function queryLayers(lng, lat, layers = ['poi', 'transit']) {
  const response = await fetch(
    `https://api.planisfy.com/v4/mapbox.places/tilequery/${lng},${lat}.json?layers=${layers.join(',')}&access_token=YOUR_KEY`
  );
  const data = await response.json();
  return data;
}
```

### Python

```python
import requests

def query_tileset(lng, lat, radius=0, tileset_id="mapbox.places"):
    """Query features from a tileset at a location."""
    url = f"https://api.planisfy.com/v4/{tileset_id}/tilequery/{lng},{lat}.json"
    params = {
        "access_token": "pk.your-key-here",
        "radius": radius,
        "limit": 10
    }

    response = requests.get(url, params=params)
    response.raise_for_status()
    return response.json()

# Point-in-polygon query
features = query_tileset(-122.4194, 37.7749)

# Radius query
nearby = query_tileset(-122.4194, 37.7749, radius=100)

# Query specific layers
url = "https://api.planisfy.com/v4/mapbox.places/tilequery/{lng},{lat}.json"
params = {
    "access_token": "pk.your-key-here",
    "layers": "poi,landuse"
}
response = requests.get(url.format(lng=-122.4194, lat=37.7749), params=params)
features = response.json()

for feature in features['features']:
    layer = feature['properties']['tilequery']['layer']
    distance = feature['properties']['tilequery']['distance']
    print(f"Layer: {layer}, Distance: {distance}m")
```

### Node.js

```javascript
import axios from 'axios';

const queryTileset = async (lng, lat, options = {}) => {
  const {
    tilesetId = 'mapbox.places',
    radius = 0,
    limit = 5,
    layers,
    geometry
  } = options;

  const params = {
    access_token: process.env.PLANISFY_API_KEY,
    ...(radius !== undefined && { radius }),
    ...(limit !== undefined && { limit }),
    ...(layers && { layers: layers.join(',') }),
    ...(geometry && { geometry })
  };

  const response = await axios.get(
    `https://api.planisfy.com/v4/${tilesetId}/tilequery/${lng},${lat}.json`,
    { params }
  );

  return response.data;
};

// Example usage
const features = await queryTileset(
  -122.4194,
  37.7749,
  {
    radius: 50,
    limit: 10,
    layers: ['poi', 'places']
  }
);

console.log(`Found ${features.features.length} features`);
```

## Response Format

### Vector Tileset Response

```json
{
  "type": "FeatureCollection",
  "features": [
    {
      "type": "Feature",
      "id": "poi_12345",
      "geometry": {
        "type": "Point",
        "coordinates": [-122.4194, 37.7749]
      },
      "properties": {
        "name": "Golden Gate Park",
        "category": "park",
        "tilequery": {
          "distance": 15.5,
          "geometry": "polygon",
          "layer": "places"
        }
      }
    }
  ]
}
```

### Response Properties

| Property | Type | Description |
|----------|------|-------------|
| `type` | `string` | Always `"Feature"` |
| `id` | `string` | The feature identifier from the tileset |
| `geometry.type` | `string` | Always `"Point"` - returns closest point on feature |
| `geometry.coordinates` | `array` | `[longitude, latitude]` coordinates of result |
| `properties` | `object` | Feature properties from tileset |
| `properties.tilequery.distance` | `number` | Distance from query point in meters (0 = point-in-polygon) |
| `properties.tilequery.geometry` | `string` | Original geometry type: `point`, `linestring`, or `polygon` |
| `properties.tilequery.layer` | `string` | Tile layer name |

<Callout type="info" title="Geometry Reference">
The API returns Point geometry for all results. For polygons, this is the query point location if inside, or the closest point on the edge if outside. For lines, it's the closest interpolated point.
</Callout>

## Query Types

### Point-in-Polygon Query

Set `radius=0` (or omit the parameter) to check which polygon(s) contain a point:

```bash
curl "https://api.planisfy.com/v4/mapbox.places/tilequery/-122.4194,37.7749.json?access_token=pk.your-key-here"
```

**Behavior:**
- Returns only polygons that contain the query point
- Points directly on polygon edges are not returned
- Useful for: "what neighborhood is this?", "what borough is this?"

### Radius Query

Specify `radius` in meters to find features within a distance:

```bash
# Find POIs within 250 meters
curl "https://api.planisfy.com/v4/mapbox.places/tilequery/-122.4194,37.7749.json?radius=250&access_token=pk.your-key-here"
```

**Behavior:**
- Returns features within the specified radius
- Results sorted by proximity (closest first)
- Useful for: "what's nearby?", "find nearest restaurants"

### Layer Filtering

Query specific layers to narrow results:

```bash
# Only POI and transit layers
curl "https://api.planisfy.com/v4/mapbox.places/tilequery/-122.4194,37.7749.json?layers=poi,transit&access_token=pk.your-key-here"
```

**Benefits:**
- Faster queries (fewer layers to search)
- More relevant results
- Specify exact layer names from your tileset

### Geometry Filtering

Return only specific geometry types:

```bash
# Only polygons
curl "https://api.planisfy.com/v4/mapbox.places/tilequery/-122.4194,37.7749.json?geometry=polygon&access_token=pk.your-key-here"

# Only points
curl "https://api.planisfy.com/v4/mapbox.places/tilequery/-122.4194,37.7749.json?geometry=point&access_token=pk.your-key-here"
```

### Multiple Tilesets

Query up to 25 tilesets simultaneously:

```bash
curl "https://api.planisfy.com/v4/mapbox.places,mapbox.streets,mapbox.buildings/tilequery/-122.4194,37.7749.json?access_token=pk.your-key-here"
```

**Use case:** Combine results from different data sources

### Deduplication

Control duplicate feature removal with `dedupe`:

```bash
# Keep all features (even duplicates)
curl "https://api.planisfy.com/v4/mapbox.places/tilequery/-122.4194,37.7749.json?dedupe=false&access_token=pk.your-key-here"

# Remove duplicates (default)
curl "https://api.planisfy.com/v4/mapbox.places/tilequery/-122.4194,37.7749.json?dedupe=true&access_token=pk.your-key-here"
```

**Deduplication criteria:** Same layer + geometry type + ID/properties

## Use Cases

### 1. "What's Here?" - Reverse Geocoding Enhancement

```javascript
// User clicks on map
map.on('click', async (e) => {
  const { lng, lat } = e.lngLat;

  // Get containing polygon
  const response = await fetch(
    `https://api.planisfy.com/v4/mapbox.places/tilequery/${lng},${lat}.json?layers=neighborhoods,boroughs&access_token=YOUR_KEY`
  );
  const data = await response.json();

  if (data.features.length > 0) {
    const neighborhood = data.features[0].properties.name;
    showPopup(`You're in ${neighborhood}`);
  }
});
```

### 2. Nearby Features Search

```javascript
// Find nearby POIs
async function findNearby(lng, lat) {
  const response = await fetch(
    `https://api.planisfy.com/v4/mapbox.places/tilequery/${lng},${lat}.json?radius=200&limit=10&layers=poi&access_token=YOUR_KEY`
  );
  const data = await response.json();

  return data.features.map(f => ({
    name: f.properties.name,
    distance: f.properties.tilequery.distance,
    category: f.properties.category
  }));
}
```

### 3. Custom Data Augmentation

```javascript
// Check if point is in your custom polygons
async function checkServiceArea(lng, lat) {
  const response = await fetch(
    `https://api.planisfy.com/v4/your.custom.delivery_zones/tilequery/${lng},${lat}.json?access_token=YOUR_KEY`
  );
  const data = await response.json();

  return data.features.length > 0;
}
```

### 4. Property Lookup

```javascript
// Get parcel/property information
async function getPropertyInfo(lng, lat) {
  const response = await fetch(
    `https://api.planisfy.com/v4/your.custom.parcels/tilequery/${lng},${lat}.json?geometry=polygon&access_token=YOUR_KEY`
  );
  const data = await response.json();

  if (data.features.length > 0) {
    return data.features[0].properties;
  }
  return null;
}
```

## Caching

Tilequery responses are cached for performance:

| Cache Type | Duration | Description |
|------------|----------|-------------|
| **Browser/Client** | 12 hours | Per `Cache-Control: max-age=43200` |
| **CDN/Edge** | 5 minutes | Per `Cache-Control: s-maxage=300` |
| **Origin** | 12 hours | Planisfy servers |

<Callout type="info" title="Cache Key">
Cache keys include coordinates, tileset ID, radius, layers, and geometry parameters. Different layer/radius combinations create different cache entries.
</Callout>

| Plan | Requests per Minute | Requests per Day |
|------|---------------------|------------------|
| Free | 100 | 5,000 |
| Starter | 300 | 30,000 |
| Pro | 600 | 100,000 |

## Error Codes

| HTTP Code | Error | Description | Solution |
|-----------|-------|-------------|------------|
| `200` | Empty FeatureCollection | No features found at location; verify layers exist |
| `400` | Bad Request | Invalid parameters; check coordinates and radius |
| `401` | Unauthorized | Missing or invalid API key |
| `403` | Forbidden | Account issue or URL restrictions on token |
| `404` | Not Found | Tileset doesn't exist or no tile at location |
| `422` | Unprocessable Entity | Invalid parameter value |
| `429` | Too Many Requests | Rate limit exceeded |
| `500` | Server Error | Internal server error |

### Error Response Format

```json
{
  "message": "Tileset mapbox.places does not exist",
  "code": "TilesetNotFound",
  "request_id": "req_abc123xyz"
}
```

### Common Errors

**"Tileset does not exist"**
```
Check tileset ID in URL. Use `mapbox.places` for hosted tilesets.
```

**"Invalid lon/lat value"**
```
Ensure coordinates are valid:
- Longitude: -180 to 180
- Latitude: -90 to 90
```

**"options.limit must be between 1 and 50"**
```
Reduce limit parameter to 50 or less.
```

## Switching from Mapbox

Replace your Mapbox tilequery URL:

### Before (Mapbox)

```javascript
const url = `https://api.mapbox.com/v4/${tilesetId}/tilequery/${lng},${lat}.json?radius=${radius}&access_token=${mapboxToken}`;
```

### After (Planisfy)

```javascript
const url = `https://api.planisfy.com/v4/${tilesetId}/tilequery/${lng},${lat}.json?radius=${radius}&access_token=${planisfyKey}`;
```

### URL Pattern Changes

| Component | Mapbox | Planisfy |
|------------|-----------|------------|
| Domain | `api.mapbox.com` | `api.planisfy.com` |
| Path | Same (`/v4/{tileset_id}/tilequery/{lon},{lat}.json`) | Same |
| Parameters | Same | Same |
| Response format | GeoJSON FeatureCollection | GeoJSON FeatureCollection |

<Callout type="tip" title="Drop-in Replacement">
Planisfy Tilequery API uses the same endpoint structure, parameters, and response format as Mapbox Tilequery v1. No code changes required beyond URL.
</Callout>

## Best Practices

### 1. Use Appropriate Radius Values

```javascript
// ❌ Too large (slow, many irrelevant results)
const radius = 5000; // 5km

// ✅ Appropriate for use case
const radius = 100; // 100m for nearby POIs
```

### 2. Specify Layers for Performance

```javascript
// ❌ Searches all layers (slower)
const url = `/tilequery/${lng},${lat}.json`;

// ✅ Searches only relevant layers (faster)
const url = `/tilequery/${lng},${lat}.json?layers=poi,places`;
```

### 3. Handle Empty Results

```javascript
const response = await fetch(url);
const data = await response.json();

if (data.features.length === 0) {
  // No features found - inform user
  showNotification('No features found at this location');
  return;
}
```

### 4. Cache Results Locally

```javascript
const tilequeryCache = new Map();

async function cachedQuery(lng, lat, radius) {
  const key = `${lng},${lat},${radius}`;

  if (tilequeryCache.has(key)) {
    return tilequeryCache.get(key);
  }

  const result = await fetchTilequery(lng, lat, radius);
  tilequeryCache.set(key, result);
  return result;
}
```

### 5. Use for Client-Side Filtering

```javascript
// Fetch larger area, filter client-side
const features = await queryTilequery(lng, lat, {
  radius: 200,
  limit: 50
});

// Filter by properties
const restaurants = features.filter(f =>
  f.properties.category === 'restaurant'
);

const openNow = features.filter(f =>
  f.properties.opening_hours === 'open'
);
```
