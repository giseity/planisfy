# Sprites

Published style sprite paths are reserved under the style URL namespace:

```text
GET /styles/v1/{owner}/{style}/sprite*
```

Current implementation returns:

```json
{ "error": { "code": "NOT_FOUND", "message": "Sprite not configured" } }
```

Sprite publication is not implemented. Do not depend on sprite responses in production styles unless you host sprite assets separately and reference them directly from the style JSON.
