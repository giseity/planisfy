# Planisfy Documentation

Planisfy is a geospatial API platform for publishing MapLibre styles, vector tiles, and map service APIs on open geospatial engines. Use it to serve public map assets, protect service APIs with API keys, and operate the same stack in managed or self-hosted environments.

## What Planisfy Serves

- Published MapLibre style JSON at `/styles/v1/{owner}/{style}` and immutable style versions at `/styles/v1/{owner}/{style}@{version}`.
- Published TileJSON and vector tiles under `/tiles/v1/*`, including dotted aliases such as `/tiles/v1/{owner}.{tileset}.json`.
- MapLibre glyph PBFs under `/fonts/v1/{fontstack}/{range}.pbf`.
- Optional style sprites under `/styles/v1/{owner}/{style}/sprite*`.
- Pelias-backed geocoding, Valhalla-backed routing, elevation lookups, and static PNG map rendering.
- Console workflows for styles, tilesets, API keys, usage, operations, and self-host checks.

## Access Model

Public published map assets can be loaded anonymously by browsers and native map clients. Authenticated requests can still be used for owner access and metering.

Service APIs such as geocoding, directions, matrix, elevation, and static maps require `X-API-Key` or an authenticated Console session. Start with [authentication](/docs/guides/authentication), then review [rate limits](/docs/guides/rate-limits) and [error handling](/docs/guides/error-handling).

## Deployment Choices

Managed Planisfy operates the provider configuration, hosted billing, email, storage, and production runtime policy for you. Self-hosted Planisfy uses the same API shapes but makes you responsible for Postgres, Redis, object storage, geospatial engines, backups, upgrades, and observability.

Read [deployment modes](/docs/concepts/deployment-modes) before choosing an environment, or go directly to [Docker Compose](/docs/self-hosting/docker) for local self-host setup.
