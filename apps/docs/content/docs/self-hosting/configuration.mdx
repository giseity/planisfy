# Configuration Guide

This guide provides detailed configuration options for all Planisfy self-hosted components.

---

## Martin Configuration

### Basic Configuration File

`config/martin/martin-config.yaml`:

```yaml
# Tile sources
sources:
  # PMTiles source
  overture:
    type: pmtiles
    path: /data/overture.pmtiles

  # PostgreSQL source
  postgres:
    type: postgresql
    connection_string: postgresql://planisfy:password@postgres:5432/tiles

  # MBTiles source
  legacy:
    type: mbtiles
    path: /data/legacy.mbtiles

# Serving options
server:
  listen_addresses: "0.0.0.0:3000"
  worker_threads: 4
  keep_alive: 75

# Cache settings
cache:
  type: redis  # or "memory"
  redis_url: redis://redis:6379

# Sprite and glyph paths
sprites:
  planisfy:
    path: /data/sprites/planisfy

glyphs:
  planisfy:
    path: /data/glyphs/planisfy
```

### Source Configuration

#### PMTiles Source

```yaml
sources:
  my_tiles:
    type: pmtiles
    path: /data/mytiles.pmtiles
```

**Parameters:**
- `path`: Path to PMTiles file
- `tile_cache_size`: Cache size in bytes

#### PostgreSQL Source

```yaml
sources:
  postgis:
    type: postgresql
    connection_string: postgresql://user:pass@host:5432/db
    tile_extent: -180,-85.0511,180,85.0511
    max_zoom: 14
    min_zoom: 0
    tile_resolution: 4096
```

**Parameters:**
- `connection_string`: PostgreSQL connection URL
- `tile_extent`: Bounding box for tiles
- `max_zoom`/`min_zoom`: Zoom level range
- `tile_resolution`: Tile size in pixels

#### MBTiles Source

```yaml
sources:
  mbtiles:
    type: mbtiles
    path: /data/tiles.mbtiles
```

### Advanced Configuration

```yaml
# Tile filtering
filters:
  exclude:
    - layer: "water"
      zoom:
        less_than: 8

# CORS settings
cors:
  allowed_origins:
    - "https://yourapp.com"
    - "https://*.yourapp.com"
  allowed_methods:
    - GET
    - HEAD
    - OPTIONS

# Rate limiting per IP
rate_limits:
  default:
    requests_per_minute: 1000
    requests_per_hour: 10000

# Compression
compression: true
compression_level: 6
```

## Authentication Configuration

### API Gateway Configuration

`config/gateway/gateway.yaml`:

```yaml
# API keys
api_keys:
  public_key: "pk.your-public-key-here"
  secret_keys:
    - "sk.your-secret-key-here"

# Allowed origins
cors:
  allowed_origins:
    - "https://yourapp.com"
    - "http://localhost:3000"

# Rate limiting
rate_limits:
  default:
    requests_per_minute: 1000
    requests_per_day: 50000
  pro:
    requests_per_minute: 10000
    requests_per_day: 500000

# Service URLs
services:
  martin: "http://martin:3000"
  pelias: "http://pelias:3100"
  valhalla: "http://valhalla:8002"
```

### API Key Middleware

```javascript
// Node.js Express middleware
const rateLimit = require('express-rate-limit');

const keyLimiter = rateLimit({
  windowMs: 60 * 1000, // 1 minute
  max: 1000, // requests per window
  keyGenerator: (req) => req.headers['x-api-key']
});

function validateApiKey(req, res, next) {
  const apiKey = req.headers['x-api-key'] || req.query.access_token;

  if (!apiKey) {
    return res.status(401).json({ error: 'Missing API key' });
  }

  if (!config.api_keys.includes(apiKey)) {
    return res.status(403).json({ error: 'Invalid API key' });
  }

  req.apiKey = apiKey;
  next();
}
```

### No Authentication (Internal)

For internal-only deployments:

```yaml
# Disable authentication
auth:
  enabled: false

# Or use IP whitelisting
auth:
  type: "ip_whitelist"
  allowed_ips:
    - "10.0.0.0/8"
    - "192.168.0.0/16"
```

## CORS Configuration

### Martin CORS

```yaml
# martin-config.yaml
cors:
  allowed_origins:
    - "https://yourapp.com"
    - "https://*.yourapp.com"
  allowed_methods:
    - GET
    - HEAD
    - OPTIONS
  allowed_headers:
    - "Content-Type"
    - "X-Requested-With"
  max_age: 86400
```

### Gateway CORS

```yaml
# gateway-config.yaml
cors:
  allowed_origins:
    - "https://yourapp.com"
    - "http://localhost:3000"
  allowed_methods:
    - GET
    - POST
    - OPTIONS
  allowed_headers:
    - "Content-Type"
    - "Authorization"
  exposed_headers:
    - "X-RateLimit-Limit"
    - "X-RateLimit-Remaining"
  max_age: 86400
  allow_credentials: true
```

## Security Settings

### TLS/SSL Configuration

```yaml
# gateway-config.yaml
tls:
  enabled: true
  cert_path: /certs/cert.pem
  key_path: /certs/key.pem
  min_version: TLSv1.2
  cipher_suites:
    - "TLS_ECDHE_RSA_WITH_AES_256_GCM_SHA384"
    - "TLS_ECDHE_RSA_WITH_AES_256_CBC_SHA384"
```

### Firewall Rules

```yaml
# Only allow necessary ports
firewall:
  allow:
    - port: 80    # HTTP
      proto: tcp
    - port: 443   # HTTPS
      proto: tcp
    - port: 22    # SSH (internal only)
      proto: tcp
      source: 10.0.0.0/8
```

### Database Security

```yaml
# PostgreSQL
postgres:
  environment:
    - POSTGRES_HOST_AUTH_METHOD=scram-sha-256
    - POSTGRES_PASSWORD_FILE=/run/secrets/postgres-password
  secrets:
    postgres_password:
      file: ./secrets/postgres_password.txt
```

## Testing Configuration

### Validate Martin Config

```bash
docker run --rm -v $(pwd)/config:/config maplibre/martin \
  martin --config /config/martin-config.yaml --mode validate
```

### Test Pelias Config

```bash
docker exec pelias \
  node -e "console.log(require('./config/pelias.json'))"
```

### Test Valhalla Config

```bash
docker exec valhalla \
  valhalla_config --config /config/valhalla/valhalla.json --validate
```

Planisfy health and preflight checks also submit a tiny route request to
Valhalla. If your self-host graph does not cover the default Stuttgart probe,
set `VALHALLA_READINESS_ROUTE=lon,lat;lon,lat` to coordinates inside your
mounted graph.

## Common Configuration Patterns

### Multiple Tile Sources

```yaml
# martin-config.yaml
sources:
  overture:
    type: pmtiles
    path: /data/overture.pmtiles

  custom:
    type: postgresql
    connection_string: postgresql://user:pass@host:5432/custom
```

### Proxy Configuration

```yaml
# For environments behind proxy
http_proxy: "http://proxy.example.com:8080"
https_proxy: "http://proxy.example.com:8080"
no_proxy: "localhost,127.0.0.1,internal.example.com"
```

### Cache Configuration

```yaml
# Redis cache
cache:
  type: redis
  redis_url: redis://redis:6379
  ttl: 3600

# Memory cache
cache:
  type: memory
  max_size: 1000
  ttl: 600
```
