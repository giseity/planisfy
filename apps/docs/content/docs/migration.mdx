### Update Your Code

**Before (Mapbox GL JS):**

```javascript
import mapboxgl from "mapbox-gl";

mapboxgl.accessToken = "YOUR_MAPBOX_KEY";
const map = new mapboxgl.Map({
  container: "map",
  style: "mapbox://styles/mapbox/streets-v12",
});
```

**After (MapLibre GL JS + Planisfy):**

```javascript
import maplibregl from "maplibre-gl";

const map = new maplibregl.Map({
  container: "map",
  style: "https://api.planisfy.com/styles/v1/planisfy/streets",
});
```

<Callout type="tip" title="Library Compatibility">
  Planisfy works with both Mapbox GL JS and MapLibre GL JS. However, MapLibre GL
  JS is open-source and recommended.
</Callout>

## Step 2: Update Style URLs

Mapbox and Planisfy use different style URL formats. Here's how to convert them:

### Style URL Format

| Component    | Mapbox                          | Planisfy                              |
| ------------ | ------------------------------- | ------------------------------------- |
| **Base URL** | `mapbox://styles/`              | `https://api.planisfy.com/styles/v1/` |
| **Username** | `mapbox` or your username       | `planisfy`                            |
| **Style ID** | `streets-v12`, `dark-v11`, etc. | `streets`, `dark`, etc.               |

### Common Style Mappings

| Mapbox Style                          | Planisfy Equivalent | URL                                                   |
| ------------------------------------- | ------------------- | ----------------------------------------------------- |
| `mapbox://styles/mapbox/streets-v12`  | `streets`           | `https://api.planisfy.com/styles/v1/planisfy/streets` |
| `mapbox://styles/mapbox/outdoors-v12` | `basic`             | `https://api.planisfy.com/styles/v1/planisfy/basic`   |
| `mapbox://styles/mapbox/dark-v11`     | `dark`              | `https://api.planisfy.com/styles/v1/planisfy/dark`    |
| `mapbox://styles/mapbox/light-v11`    | _(use `basic`)_     | `https://api.planisfy.com/styles/v1/planisfy/basic`   |

### Custom Styles

If you have custom Mapbox styles, you'll need to:

1. **Export your style JSON** from Mapbox Studio
2. **Replace sprite URLs** in the JSON:

   ```json
   // Before
   "sprite": "mapbox://sprites/mapbox/your-style"

   // After
   "sprite": "https://api.planisfy.com/sprites/v1/planisfy/your-style"
   ```

3. **Replace glyph URLs**:

   ```json
   // Before
   "glyphs": "mapbox://fonts/mapbox/{fontstack}/{range}.pbf"

   // After
   "glyphs": "https://api.planisfy.com/glyphs/v1/planisfy/{fontstack}/{range}.pbf"
   ```

4. **Replace tile sources**:

   ```json
   // Before
   "tiles": ["https://api.mapbox.com/v4/mapbox.mapbox-streets-v8/{z}/{x}/{y}.vector.pbf?access_token=YOUR_KEY"]

   // After
   "tiles": ["https://api.planisfy.com/tiles/v1/streets/{z}/{x}/{y}.mvt"]
   ```

## Step 3: Update API Endpoints

### Geocoding API

**Before (Mapbox):**

```javascript
const response = await fetch(
  `https://api.mapbox.com/geocoding/v5/mapbox.places/${encodeURIComponent(query)}.json?access_token=${accessToken}`,
);
```

**After (Planisfy):**

```javascript
const response = await fetch(
  `https://api.planisfy.com/geocoding/v1/search?q=${encodeURIComponent(query)}`,
);
```

<Callout type="warn" title="Geocoding Status">
  The Planisfy Geocoding API is <strong>currently in development</strong>. Check
  the <a href="/docs/api/geocoding">Geocoding API</a> documentation for the
  latest status.
</Callout>

### Directions API

**Before (Mapbox):**

```javascript
const response = await fetch(
  `https://api.mapbox.com/directions/v5/mapbox/driving/${startLng},${startLat};${endLng},${endLat}?access_token=${accessToken}`,
);
```

**After (Planisfy):**

```javascript
const response = await fetch(
  `https://api.planisfy.com/directions/v1/route?profile=driving&points=${startLng},${startLat}|${endLng},${endLat}`,
);
```

<Callout type="warn" title="Directions Status">
  The Planisfy Directions API is <strong>currently in development</strong>.
  Check the <a href="/docs/api/directions">Directions API</a> documentation for
  the latest status.
</Callout>

## Step 4: Replace API Keys

### Environment Variables

Update your `.env` file:

```bash
# Remove
# VITE_MAPBOX_TOKEN=pk.your-mapbox-key

# Add
VITE_PLANISFY_TOKEN=pk_your_planisfy_key
```

### Code Updates

**Before:**

```javascript
const accessToken = import.meta.env.VITE_MAPBOX_TOKEN;
```

**After:**

```javascript
const accessToken = import.meta.env.VITE_PLANISFY_TOKEN;
```

## Step 5: Framework-Specific Migration

### React

**Before (react-map-gl with Mapbox):**

```javascript
import Map from "react-map-gl";

<Map
  mapboxAccessToken={MAPBOX_TOKEN}
  initialViewState={{ longitude: -74.006, latitude: 40.7128, zoom: 12 }}
  mapStyle="mapbox://styles/mapbox/streets-v12"
/>;
```

**After (react-map-gl with MapLibre + Planisfy):**

```javascript
import Map from "react-map-gl/maplibre";

<Map
  initialViewState={{ longitude: -74.006, latitude: 40.7128, zoom: 12 }}
  mapStyle={`https://api.planisfy.com/styles/v1/planisfy/streets`}
/>;
```

<Callout type="info" title="React Map GL v8+">
  <a href="https://visgl.github.io/react-map-gl/docs/">react-map-gl v8.0+</a>{" "}
  (released January 2025) supports both Mapbox GL JS and MapLibre GL JS. Use the{" "}
  <code>@maplibre</code> variant.
</Callout>

### Vue

**Before (vue-mapbox):**

```vue
<template>
  <MapboxMap
    :access-token="MAPBOX_TOKEN"
    map-style="mapbox://styles/mapbox/streets-v12"
  />
</template>
```

**After (vue-maplibre-gl):**

```vue
<template>
  <MapLibreMap
    :map-style="`https://api.planisfy.com/styles/v1/planisfy/streets`"
  />
</template>
```

### Next.js

**Before:**

```javascript
import { useEffect } from "react";
import mapboxgl from "mapbox-gl";

export default function Map() {
  useEffect(() => {
    mapboxgl.accessToken = MAPBOX_TOKEN;
    new mapboxgl.Map({
      container: "map",
      style: "mapbox://styles/mapbox/streets-v12",
    });
  }, []);
  return <div id="map" />;
}
```

**After:**

```javascript
"use client";

import { useEffect } from "react";
import maplibregl from "maplibre-gl";
import "maplibre-gl/dist/maplibre-gl.css";

export default function Map() {
  useEffect(() => {
    new maplibregl.Map({
      container: "map",
      style: `https://api.planisfy.com/styles/v1/planisfy/streets`,
    });
  }, []);
  return <div id="map" />;
}
```

## Step 6: Handle Breaking Differences

While Planisfy aims for compatibility, there are some differences to be aware of:

### Style Specification Differences

Mapbox and MapLibre have diverged slightly in the style spec. Common issues:

| Feature               | Mapbox        | Planisfy/MapLibre | Notes                               |
| --------------------- | ------------- | ----------------- | ----------------------------------- |
| **Expression syntax** | Latest        | 1.x based         | Some newer expressions may not work |
| **Custom layers**     | `custom` type | Not supported     | Use standard layer types            |
| **3D terrain**        | Supported     | Limited           | Basic 3D only                       |

### Font Stack Differences

Mapbox uses proprietary fonts. Planisfy uses open-source alternatives:

| Mapbox Font | Planisfy Equivalent    |
| ----------- | ---------------------- |
| `DIN Pro`   | `Open Sans` / `Roboto` |
| `Mark Offc` | `Montserrat` / `Lato`  |

If you notice text rendering differences, update your `text-font` property in the style JSON.

## Step 7: Test Your Migration

After making changes, test your application:

1. **Visual Check**: Verify map renders correctly at multiple zoom levels
2. **Interactions**: Test zoom, pan, rotation
3. **Markers/Popups**: Ensure all markers and popups appear
4. **Custom Layers**: If using custom data layers, verify they render
5. **API Calls**: Test any direct API calls (geocoding, directions, etc.)

<Callout type="tip" title="Browser DevTools">
  Use the browser's Network tab to inspect API requests. Verify that requests go
  to <code>api.planisfy.com</code> and responses are successful.
</Callout>
