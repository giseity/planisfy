# Caching Strategies Guide

This guide explains how to use caching effectively to improve performance, reduce API costs, and provide a better user experience.

---

## Why Cache?

Caching provides three key benefits:

### 1. Performance

**Reduce latency** by serving responses from local cache:

| Source | Typical Response Time |
|--------|----------------------|
| CDN cache | 20-50ms |
| Browser cache | \<10ms |
| API request | 100-500ms+ |

### 2. Cost Reduction

**Reduce API calls** to stay within quotas:

- Tiles: Cache for 12 hours → 50% fewer requests
- Styles: Cache for 1 hour → 90% fewer requests
- Glyphs: Cache for 1 year → 99% fewer requests

### 3. User Experience

**Faster map loading** and offline capability:

- Instant map display on repeat visits
- Smooth zooming/panning with cached tiles
- Offline maps for mobile apps

## Cache Headers

Planisfy uses standard HTTP cache headers to control caching behavior.

### Response Cache Headers

| Header | Description | Example |
|--------|-------------|---------|
| `Cache-Control` | Who can cache and for how long | `public, max-age=43200` |
| `ETag` | Unique identifier for response version | `"33a64df551425fcc55e4d42a148795d9f25f89d4"` |
| `Last-Modified` | When the resource last changed | `Wed, 21 Oct 2015 07:28:00 GMT` |
| `Expires` | Absolute expiration time | `Fri, 30 Dec 2025 12:00:00 GMT` |
| `Vary` | What varies the response | `Vary: Accept-Encoding` |

### Understanding Cache-Control

```
Cache-Control: public, max-age=43200, must-revalidate
```

| Directive | Meaning |
|-----------|---------|
| `public` | Response can be cached by browsers and CDNs |
| `private` | Only cache in browser (not CDN) |
| `max-age=N` | Cache is fresh for N seconds |
| `must-revalidate` | Must check with server after max-age |
| `no-cache` | Can be cached but must revalidate before use |
| `no-store` | Do not cache at all |

## Client-Side Caching

### Browser HTTP Cache

The browser automatically caches based on `Cache-Control` headers. No code needed!

```javascript
// Browser caches automatically based on headers
fetch('https://api.planisfy.com/styles/v1/planisfy/basic?access_token=YOUR_KEY');
// Subsequent requests use browser cache until max-age expires
```

### Service Worker Cache

For progressive web apps (PWAs):

```javascript
// Install event - cache static resources
self.addEventListener('install', (event) => {
  event.waitUntil(
    caches.open('planisfy-v1').then((cache) => {
      return cache.addAll([
        'https://api.planisfy.com/styles/v1/planisfy/basic?access_token=YOUR_KEY'
      ]);
    })
  );
});

// Fetch event - serve from cache, fall back to network
self.addEventListener('fetch', (event) => {
  event.respondWith(
    caches.match(event.request).then((response) => {
      return response || fetch(event.request);
    })
  );
});
```

### Application Cache (MapLibre GL JS)

MapLibre automatically caches tiles:

```javascript
const map = new maplibregl.Map({
  container: 'map',
  style: 'https://api.planisfy.com/styles/v1/planisfy/basic?access_token=YOUR_KEY',
  // MapLibre automatically caches tiles in browser
  crossOrigin: 'anonymous'
});
```

To clear the cache:

```javascript
// Clear specific tile
map.style.sourceCaches['planisfy'].clearTiles();

// Clear all cached data
map.style._requestManager.clear();
```

## Conditional Requests

Use ETags to avoid downloading unchanged resources:

### How ETags Work

1. **First request:** Server returns `ETag: "abc123"`
2. **Second request:** Client sends `If-None-Match: "abc123"`
3. **Server responds:** `304 Not Modified` (no body)

**Bandwidth saved:** 99%+ (headers only)

### Implementing Conditional Requests

```javascript
let cachedETag = null;

async function getStyleWithETag(styleId) {
  const headers = {};
  if (cachedETag) {
    headers['If-None-Match'] = cachedETag;
  }

  const response = await fetch(
    `https://api.planisfy.com/styles/v1/planisfy/${styleId}?access_token=YOUR_KEY`,
    { headers }
  );

  // Save ETag from response
  const newETag = response.headers.get('ETag');
  if (newETag) {
    cachedETag = newETag;
  }

  if (response.status === 304) {
    return cachedStyle; // Use cached version
  }

  // Cache and return new style
  cachedStyle = await response.json();
  return cachedStyle;
}
```

### Conditional Request Benefits

| Scenario | Without ETag | With ETag |
|----------|--------------|-----------|
| **Unchanged style** | 50 KB download | 500 B (headers only) |
| **Changed style** | 50 KB download | 50 KB download |
| **Bandwidth saved** | 0% | 99% for unchanged |

## CDN Caching

Planisfy uses Cloudflare CDN for edge caching:

### How CDN Caching Works

```
User → Cloudflare Edge → Planisfy Origin
         ↓ (cache hit)
      Response
```

**Benefits:**
- Requests served from nearest edge location
- Reduced origin load
- Faster response times

### CDN Cache Levels

| Cache Level | Duration | Description |
|-------------|----------|-------------|
| **Edge** | 5 minutes - 7 days | Cloudflare edge servers worldwide |
| **Browser** | 5 min - 1 year | User's browser cache |

### Cache Purging

CDN caches automatically refresh when:
- `max-age` expires
- Manual purge via API (self-hosted only)

<Callout type="tip" title="Cache Headers Apply Automatically">
Planisfy sets appropriate cache headers on all responses. Your browser and CDN will automatically cache based on these headers.
</Callout>

## Application-Level Caching

### In-Memory Cache

Cache frequently used data in memory:

```javascript
class MemoryCache {
  constructor(ttl = 60000) {
    this.cache = new Map();
    this.ttl = ttl;
  }

  set(key, value) {
    this.cache.set(key, {
      value,
      expires: Date.now() + this.ttl
    });
  }

  get(key) {
    const item = this.cache.get(key);

    if (!item) return null;

    if (Date.now() > item.expires) {
      this.cache.delete(key);
      return null;
    }

    return item.value;
  }
}

// Usage
const styleCache = new MemoryCache(60 * 60 * 1000); // 1 hour

async function getCachedStyle(styleId) {
  let style = styleCache.get(styleId);

  if (!style) {
    style = await fetchStyle(styleId);
    styleCache.set(styleId, style);
  }

  return style;
}
```

### LocalStorage Cache

Persist cache across sessions:

```javascript
class LocalStorageCache {
  constructor(prefix = 'planisfy_') {
    this.prefix = prefix;
  }

  set(key, value, maxAge = 3600000) {
    const item = {
      value,
      expires: Date.now() + maxAge
    };
    localStorage.setItem(this.prefix + key, JSON.stringify(item));
  }

  get(key) {
    const item = localStorage.getItem(this.prefix + key);

    if (!item) return null;

    const parsed = JSON.parse(item);

    if (Date.now() > parsed.expires) {
      localStorage.removeItem(this.prefix + key);
      return null;
    }

    return parsed.value;
  }

  clear() {
    Object.keys(localStorage)
      .filter(key => key.startsWith(this.prefix))
      .forEach(key => localStorage.removeItem(key));
  }
}

// Usage
const cache = new LocalStorageCache();

cache.set('style-basic', styleData, 60 * 60 * 1000); // 1 hour
const cached = cache.get('style-basic');
```

<Callout type="warn" title="LocalStorage Limits">
LocalStorage has a 5-10 MB limit. Don't cache large datasets like tile collections.
</Callout>

### IndexedDB Cache

For larger datasets (tiles, offline maps):

```javascript
class IndexedDBCache {
  constructor(dbName = 'PlanisfyCache', storeName = 'resources') {
    this.dbName = dbName;
    this.storeName = storeName;
  }

  async init() {
    return new Promise((resolve, reject) => {
      const request = indexedDB.open(this.dbName, 1);

      request.onerror = () => reject(request.error);
      request.onsuccess = () => resolve(request.result);

      request.onupgradeneeded = (event) => {
        const db = event.target.result;
        if (!db.objectStoreNames.contains(this.storeName)) {
          db.createObjectStore(this.storeName, { keyPath: 'url' });
        }
      };
    });
  }

  async put(url, data, maxAge = 3600000) {
    const db = await this.init();

    return new Promise((resolve, reject) => {
      const transaction = db.transaction([this.storeName], 'readwrite');
      const store = transaction.objectStore(this.storeName);

      const request = store.put({
        url,
        data,
        expires: Date.now() + maxAge
      });

      request.onsuccess = () => resolve();
      request.onerror = () => reject(request.error);
    });
  }

  async get(url) {
    const db = await this.init();

    return new Promise((resolve, reject) => {
      const transaction = db.transaction([this.storeName], 'readonly');
      const store = transaction.objectStore(this.storeName);
      const request = store.get(url);

      request.onsuccess = () => {
        const result = request.result;

        if (!result) {
          resolve(null);
          return;
        }

        if (Date.now() > result.expires) {
          this.delete(url);
          resolve(null);
          return;
        }

        resolve(result.data);
      };

      request.onerror = () => reject(request.error);
    });
  }

  async delete(url) {
    const db = await this.init();

    return new Promise((resolve, reject) => {
      const transaction = db.transaction([this.storeName], 'readwrite');
      const store = transaction.objectStore(this.storeName);
      const request = store.delete(url);

      request.onsuccess = () => resolve();
      request.onerror = () => reject(request.error);
    });
  }
}
```

## Offline Caching

### PWA Offline Support

Make maps work offline:

```javascript
// Cache tiles for offline use
self.addEventListener('install', (event) => {
  event.waitUntil(
    caches.open('offline-tiles-v1').then((cache) => {
      // Pre-cache viewport tiles
      const tiles = generateTileList(center, zoom, 10, 10);

// Serve tiles from cache when offline
self.addEventListener('fetch', (event) => {
```

## Cache Invalidation

### Manual Invalidation

Clear caches when needed:

```javascript
// Clear specific cache
localStorage.removeItem('planisfy_style-basic');

// Clear all Planisfy caches
Object.keys(localStorage)
  .filter(key => key.startsWith('planisfy_'))
  .forEach(key => localStorage.removeItem(key));

// Clear MapLibre tile cache
map.style._requestManager.clear();
```

### Version-Based Invalidation

Use version numbers to invalidate caches:

```javascript
const CACHE_VERSION = 'v2';

async function getCachedStyle(styleId) {
  const cacheKey = `${CACHE_VERSION}_${styleId}`;

  let style = localStorage.getItem(cacheKey);

  if (!style) {
    style = await fetchStyle(styleId);
    localStorage.setItem(cacheKey, JSON.stringify(style));
  }

  return JSON.parse(style);
}

// When you update the app, increment CACHE_VERSION
// All old caches will be automatically bypassed
```

### Smart Invalidation

Invalidate only changed resources:

```javascript
async function updateStyleIfNeeded(styleId, currentETag) {
  const response = await fetch(
    `https://api.planisfy.com/styles/v1/planisfy/${styleId}?access_token=YOUR_KEY`,
    {
      headers: {
        'If-None-Match': currentETag
      }
    }
  );

  if (response.status === 304) {
    return false; // Not modified
  }

  const newStyle = await response.json();
  const newETag = response.headers.get('ETag');

  return { style: newStyle, etag: newETag };
}
```

### 2. Preload Tiles

Load tiles before user interaction:

```javascript
map.on('load', () => {
  // Preload adjacent zoom levels
  for (let z = 10; z <= 14; z++) {
    map.triggerRepaint();
  }
});
```

### 3. Limit Cache Size

Prevent cache from growing too large:

```javascript
const MAX_CACHE_SIZE = 50 * 1024 * 1024; // 50 MB

function checkCacheSize() {
  if (navigator.storage && navigator.storage.estimate) {
    navigator.storage.estimate().then((estimate) => {
      const usage = estimate.usage;
      const quota = estimate.quota;

      if (usage > MAX_CACHE_SIZE) {
        clearOldestCacheEntries();
      }
    });
  }
}
```

### 4. Warm Up Cache

Load frequently accessed tiles:

```javascript
async function warmupCache(styleId) {
  const hotRegions = [
    { center: [-122.4194, 37.7749], zoom: 12 }, // SF

    const tiles = getTilesForRegion(region.center, region.zoom, 3, 3);

    await Promise.all(
      tiles.map(tile => fetch(tile.url))
    );
  }
}
```

## Cache Monitoring

### Track Cache Performance

Monitor cache hit rates:

```javascript
const cacheMetrics = {
  hits: 0,
  misses: 0,

  get hitRate() {
    const total = this.hits + this.misses;
    return total > 0 ? this.hits / total : 0;
  }
};

async function getCachedOrFetch(url) {
  const cached = cache.get(url);

  if (cached) {
    cacheMetrics.hits++;
    return cached;
  }

  cacheMetrics.misses++;
  const data = await fetch(url).then(r => r.json());
  cache.set(url, data);

  return data;
}

// Log cache performance
setInterval(() => {
  console.log(`Cache hit rate: ${(cacheMetrics.hitRate * 100).toFixed(1)}%`);
}, 60000); // Every minute
```

### Cache Size Monitoring

Monitor cache storage usage:

```javascript
async function getCacheSize() {
  if (navigator.storage && navigator.storage.estimate) {
    const estimate = await navigator.storage.estimate();

    return {
      usage: estimate.usage,
      quota: estimate.quota,
      usagePercent: (estimate.usage / estimate.quota * 100).toFixed(2)
    };
  }
}

// Alert when cache is full
async function checkCacheSize() {
  const size = await getCacheSize();

  if (size.usagePercent > 90) {
    console.warn('Cache is 90% full. Consider clearing old entries.');
  }
}
```

## Caching Strategies by Use Case

### Web Application

```javascript
// Service Worker for web app
self.addEventListener('fetch', (event) => {
  const url = new URL(event.request.url);

  // Cache styles, glyphs, sprites aggressively
  if (url.pathname.includes('/styles/') ||
      url.pathname.includes('/fonts/') ||
      url.pathname.includes('/sprite')) {
    event.respondWith(
      caches.match(event.request).then((response) => {
        return response || fetch(event.request).then(fetchResponse => {
          return caches.open('planisfy-v1').then(cache => {
            cache.put(event.request, fetchResponse.clone());
            return fetchResponse;
          });
        });
      })
    );
  }

  // Network-first for tiles (want freshest data)
  if (url.pathname.includes('/tiles/')) {
    event.respondWith(
      fetch(event.request).catch(() => {
        return caches.match(event.request);
      })
    );
  }
});
```

### Mobile Application

```javascript
// React Native with persistent cache
import AsyncStorage from '@react-native-async-storage/async-storage';

async function getCachedStyle(styleId) {
  try {
    const cached = await AsyncStorage.getItem(`style_${styleId}`);

    if (cached) {
      const { data, timestamp } = JSON.parse(cached);
      const age = Date.now() - timestamp;

      if (age < 3600000) { // 1 hour
        return data;
      }
    }

    // Fetch fresh data
    const fresh = await fetchStyle(styleId);
    await AsyncStorage.setItem(`style_${styleId}`, JSON.stringify({
      data: fresh,
      timestamp: Date.now()
    }));

    return fresh;
  } catch (error) {
    console.error('Cache error:', error);
    return fetchStyle(styleId);
  }
}
```

### Server-Side Application

```javascript
// Node.js with Redis cache
import Redis from 'ioredis';

const redis = new Redis();

async function getCachedStyle(styleId) {
  const cacheKey = `style:${styleId}`;

  // Try Redis cache
  const cached = await redis.get(cacheKey);

  if (cached) {
    return JSON.parse(cached);
  }

  // Fetch from API
  const style = await fetchStyle(styleId);

  // Cache for 1 hour
  await redis.setex(cacheKey, 3600, JSON.stringify(style));

  return style;
}
```

## Troubleshooting

### Cache Not Working

**Problem:** Resources are not being cached

**Solutions:**
1. Check `Cache-Control` headers in response
2. Ensure browser allows caching (not in private browsing)
3. Clear existing cache and try fresh
4. Verify no browser extensions are blocking cache

### Stale Data

**Problem:** Old data is being served

**Solutions:**
1. Add `must-revalidate` to cache headers
2. Implement cache versioning
3. Use conditional requests with ETags
4. Set appropriate `max-age`

### Cache Too Large

**Problem:** Exceeding browser storage limits

**Solutions:**
1. Implement cache size limits
2. Use LRU (least recently used) eviction
3. Store only essential data
4. Consider IndexedDB for larger datasets

## Best Practices

### Do's ✅

- **Use cache headers** provided by the API
- **Implement conditional requests** with ETags
- **Cache aggressively** for immutable resources (glyphs, sprites)
- **Set appropriate TTL** based on how often data changes
- **Monitor cache hit rates** to optimize caching strategy
- **Implement fallback** when cache is unavailable
- **Clear old cache** to prevent storage limits

### Don'ts ❌

- **Don't cache** responses with `Cache-Control: no-store`
- **Don't ignore** ETag headers
- **Don't cache** sensitive data (user locations, searches)
- **Don't cache** indefinitely without versioning
- **Don't rely** solely on browser cache for offline support
- **Don't cache** large datasets in localStorage
- **Don't forget** to clear cache when users log out
