# Quick Start Guide

This guide gets a browser map on screen using Planisfy's current published
style contract. For local self-hosting, use the seeded Planisfy Streets fixture.
For your own data, publish a style in Console and swap in its public style URL.

---

## Step 1: Choose A Style URL

Published public styles are loaded directly by MapLibre:

```text
https://api.planisfy.com/styles/v1/{owner}/{style}
https://api.planisfy.com/styles/v1/{owner}/{style}@{version}
```

For a local self-host demo after running the setup flow, use:

```text
http://localhost:4000/styles/v1/planisfy/planisfy-streets-light-v1
```

<Callout type="info" title="API keys">
  Planisfy API keys start with <code>pk_</code>. Published public map assets can
  load anonymously, while protected service APIs and private resources use the
  <code>X-API-Key</code> header.
</Callout>

<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />

      href="https://unpkg.com/maplibre-gl@4/dist/maplibre-gl.css"
      rel="stylesheet"

    <style>
      body {
        margin: 0;
      }
      #map {
        width: 100vw;
        height: 100vh;
      }
    </style>
  </head>
  <body>
    <div id="map"></div>
    <script src="https://unpkg.com/maplibre-gl@4/dist/maplibre-gl.js"></script>
    <script>
      const map = new maplibregl.Map({
        container: "map",
        style:
          "http://localhost:4000/styles/v1/planisfy/planisfy-streets-light-v1",
        center: [9.1829, 48.7758],
        zoom: 12,
      });

      map.addControl(new maplibregl.NavigationControl(), "top-right");
    </script>
  </body>
</html>

## Step 3: Verify The Style

Before wiring a style into an app, check that the published URL returns JSON:

```bash
curl "http://localhost:4000/styles/v1/planisfy/planisfy-streets-light-v1" \
  -H "Accept: application/json"
```

For protected Console and service APIs, pass the API key as a header:

```bash
curl "http://localhost:4000/geocoding/v1/search?text=Stuttgart" \
  -H "X-API-Key: pk_your_key_here"
```

<Callout type="warn" title="Local fixture data">
  The local Planisfy Streets fixture needs the Stuttgart PMTiles demo artifact.
  If the artifact is missing, the stack can boot, but the default map will
  report fixture-data readiness warnings until compatible PMTiles are supplied.
</Callout>

## Step 4: Publish Your Own Style

1. Upload or import source data in Console.
2. Build and publish a tileset.
3. Add the tileset source to a style.
4. Publish the style.
5. Use the stable URL, or pin an immutable `@version` URL for production.

```text
https://api.planisfy.com/styles/v1/{owner}/{style}
https://api.planisfy.com/styles/v1/{owner}/{style}@{version}
```
